🏗️ アーキテクチャ

設計方針

  • 各ツールは完全に自己完結した単一HTMLファイル（CSS・JS全てインライン）
  • 共通ライブラリなし → ファイル間の依存関係ゼロ
  • localStorageをシンプルなJSONストアとして使用
  • 全ての計算はクライアントサイドで完結

各ファイルの内部構造

<!-- 1. <head>: CSS変数・スタイル定義 -->

XSSエスケープ（全ツール共通）

// 全ツールのscript先頭に定義済み
function esc(s) {
  const d = document.createElement('div');
  d.textContent = String(s ?? '');
  return d.innerHTML; // HTMLエンティティ化された文字列を返す
}
// 使用例: innerHTML内でユーザーデータを展開する際は必ずesc()を通す
el.innerHTML = `<td>${esc(user.name)}</td>`;

🏠 Tool 1 — AI査定ツール

価格算出アルゴリズム

ヘドニック価格モデルをベースに7因子の積で推定価格を算出する。

price = baseUnitPrice(area) × sqm
        × ageFactor(age)          // 0.44〜1.0
        × typeFactor(propType)    // 0.65〜1.05
        × floorFactor(floor)      // 0.86〜1.20
        × stationFactor(station)  // 0.86〜1.15
        × directionFactor(dir)    // 0.93〜1.04
        × reformFactor(reform)    // 1.00〜1.10

エリア基準単価（円/㎡）の修正方法

ファイル内の AREA_UNIT オブジェクトの値を変更することで各エリアの基準単価を更新できる。

const AREA_UNIT = {
  tokyo_central: 1500000, // ← この数値を変更
  tokyo_yamate:  1200000,
  ...
};

流動性スコア計算式

liquidity = (stationFac - 0.86) / (1.15 - 0.86) × 40
          + (1 - min(age, 40) / 40) × 40
          + (dirFac - 0.93) / (1.04 - 0.93) × 20
// 合計 0〜100 でキャップ

🤝 Tool 3 — オーナーダイレクト

マッチングアルゴリズム

calcMatchScore(seller, buyer) 関数が4つの基準でスコアを算出する（最大100点）。

基準最大点条件
価格帯一致40点buyer.budget ≥ seller.price × 90%
エリア一致30点エリア文字列の部分一致（先頭4文字）
物件種別一致20点buyer.type === seller.type または "any"
面積条件10点seller.sqm ≥ buyer.sqm（希望面積以上）

スコア30点以上のペアのみ表示される。閾値を変更するには if(m.score >= 30) の値を変える。

手数料計算式

// 仲介手数料上限（宅建業法）
const baseFee =
  price <= 200  ? price × 0.05 :
  price <= 400  ? price × 0.04 + 2 :
                  price × 0.03 + 6;  // 万円単位
// 両手: × 2 、割引両手: price × 0.01 × 2

🏢 Tool 4 — 管理SaaS

管理委託費の計算ロジック

// ダッシュボードの manageFee 計算
const manageFee = properties.reduce((sum, prop) => {
  const tenants = activeTenants.filter(t => t.propertyId === prop.id);
  const rentSum = tenants.reduce((x,t) => x + (t.rent || 0), 0);
  return sum + rentSum × (prop.rate || 5) / 100;
}, 0);

アクション必要リストの条件

  • 家賃未入金：当月の家賃レコードが unpaid のテナント
  • 契約更新：契約終了日まで90日以内のアクティブテナント
  • 緊急修繕：priority === 'urgent' かつ status !== 'done' の修繕

収益グラフ（Chart.js）

グラフは renderDashboard() 呼び出し時に過去12ヶ月の saas_rents データを集計して描画する。同関数は毎回 revenueChart.destroy() してから再生成するため、タブ切替で重複描画しない。

📊 Tool 5 — 投資分析ツール

主要計算関数

calcLoanPayment(loan, rate, term, type)

// 元利均等返済の月次返済額
r = rate / 100 / 12;  // 月利
monthly = L × r × (1+r)^n / ((1+r)^n - 1);  // n = term×12

// 元金均等返済（初月最大）
firstMonth = L/(term×12) + L×r;

calcNOI(rent, mgmt, tax, ins, mrate, vacancy)

annualRent  = rent × 12 × (1 - vacancy/100);
annualMgmt  = mgmt × 12;
mrateFee    = annualRent × mrate/100;
NOI = annualRent - annualMgmt - mrateFee - tax - ins;

IRR近似式（簡易CAGR）

// ⚠️ 真のIRRではなく保有期間CAGR近似
irr = (exitNet + NOI×hold - balance) / equity)^(1/hold) - 1
開発者注意

現在のIRRはキャッシュフローの時間的価値を考慮しないCGAR近似です。真のIRRはNewton-Raphson法などで求める必要があります。将来バージョンで改善予定。

感度分析マトリクス

空室率0〜25%（6段階）× 家賃変化-20%〜+10%（6段階）の36セルを総当りで calcNOI() を呼び出し、実質利回りを計算・色分けして表示する。

🔒 セキュリティ

XSS（クロスサイトスクリプティング）対策

ユーザー入力データを innerHTML に展開する箇所は全て esc() 関数を通している。

// ✅ 安全: esc()でエスケープ
el.innerHTML = `<td>${esc(user.name)}</td>`;

// ❌ 危険: 直接展開（修正済み）
el.innerHTML = `<td>${user.name}</td>`;
適用済み

Tool 1〜5の全 innerHTML ユーザーデータ展開箇所に esc() を適用済み（v1.0時点）。

データ漏洩リスク

  • localStorageは同一オリジン（file://）からのみアクセス可能
  • 外部サーバーへのデータ送信は一切ない
  • ただし、端末の物理アクセスを得た第三者はlocalStorageを読み取れる

CDN依存のリスク

Chart.js（jsDelivr）・Google Fontsを外部CDNから読み込んでいる。信頼性は高いが以下のリスクがある。

  • オフライン時にグラフ・フォントが使えない
  • CDNサービス側の障害時に影響を受ける
完全オフライン化する場合

Chart.js（chart.umd.min.js）をダウンロードしてローカルに配置し、CDNのURLをローカルパスに変更する。Google FontsはCSS内の font-family をシステムフォントにフォールバックさせるか、フォントファイルを同梱する。

Tool 4でCSVインポート機能を追加する場合

function importCSV(file) {
  const reader = new FileReader();
  reader.onload = (e) => {
    const rows = e.target.result.split('\n').slice(1); // ヘッダ除去
    rows.forEach(row => {
      const [name, addr, ...] = row.split(',');
      tenants.push({ id: Date.now(), name, addr, ... });
    });
    save('tenants');
    renderTenants();
  };
  reader.readAsText(file, 'utf-8');
}

バックエンドAPI連携に移行する場合

現在 localStorage.getItem/setItem を呼び出している箇所を fetch() + REST API に置き換えることで、マルチデバイス対応・データ共有が実現できる。各ツールの save(key) 関数と初期化部分（let data = JSON.parse(...)）が変更対象となる。

🚀 デプロイ方針

現状（ローカル運用）

現在はローカルファイルとして運用。追加のセットアップ不要。

静的ホスティングへの公開（将来）

公開前の必須対応

運営者の指示により現在は非公開。公開する場合は以下を対応すること：

  1. 認証機能の追加（現在なし）
  2. localStorageではなくサーバーサイドDBへのデータ移行
  3. HTTPS対応（静的ホスティングサービスを使用）
  4. プライバシーポリシーの整備（個人情報取扱いの明示）
  5. 宅建業法上の表示義務の確認（AI査定の免責事項）
