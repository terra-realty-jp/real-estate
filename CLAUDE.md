# CLAUDE.md — real-estate-project 自律実装ルール

## 基本方針

このプロジェクトでは「制限に達するまで実装を回し続ける」が常時有効な指示です。
ユーザーへの確認・質問は不要。自律的に判断して実装・コミット・プッシュを続けてください。

## 実装ループ（毎セッション自動継続）

1. 現在の各ツールの状態をgrepで確認
2. まだ実装されていない機能を選ぶ（下記ルールに従う）
3. 実装 → 構文チェック → `git add` → `git commit` → `git push origin main`
4. 次の機能へ。レートリミットに達するまで繰り返す

## 実装対象ファイル

- `tools/1-ai-satei.html` — AI査定ツール
- `tools/2-akiya-hunter.html` — 空き家診断ツール
- `tools/3-owner-direct.html` — オーナーダイレクトマッチング
- `tools/4-kanri-saas.html` — 賃貸管理SaaS
- `tools/5-toushi-bunseki.html` — 不動産投資分析
- `index.html` — トップページ

## ターゲットユーザー

不動産オーナー・投資家（40〜60代・スマートフォン利用者・非専門家）

## 実装ルール

- **重複チェック必須**: 実装前に必ず `grep` で既存機能を確認し、重複しない機能を選ぶ
- **構文チェック必須**: コミット前に `node -e "..."` で JS 構文エラーがないことを確認
- **1機能ずつコミット**: 機能単位でコミットし、push まで完了させてから次へ
- **ユーザーへの質問禁止**: 機能の選択・実装方針はすべて自律判断する
- **index.html の構文チェック**: JSON-LD ブロックを除外するフィルタを使うこと
  ```
  /<script(?![^>]*type="application\/ld)[^>]*>[\s\S]*?<\/script>/g
  ```

## コミットメッセージ形式

```
Tool1: 〜機能を追加（〜の説明）
Tool2: 〜カード追加（〜）
index: 〜
```

## 実装アイデアの優先順位

1. **実用性が高い**（オーナーが日常的に使う機能）
2. **スマホで見やすい**（カード形式・大きい数字・色分け）
3. **計算・自動診断系**（入力値から自動で結果を出す）
4. **ビジュアライゼーション**（グラフ・マトリックス・タイムライン）

## 直近コミット済み機能（重複禁止）

最新の `git log --oneline -30` を確認して重複を避けること。
